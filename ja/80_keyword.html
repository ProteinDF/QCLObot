QCLObot YAML ファイル解説

QCLObot プレイブック(入力ファイル)は、以下の2つのパートに分かれています。

  • 変数(vars)セクション

    変数の定義

  • タスク(tasks)セクション

    フレーム分子に対する処理内容の定義

変数(vars)セクション

変数を宣言します。ここで定義された変数は、 with_items 構文などで利用されます。

vars:
  residues: [27, 28, 48, 97, 101, 108, 248, 547, 558, 559]

tasks:
  - name: frame_{{ item }}
    sp: true
    fragments:
      - name: AA_{{ item }}
        brd_select: /model_1/A/{{ item }}/
      - name: ACE_{{ item }}
        add_ACE: true
        brd_select: /model_1/A/{{ item -1 }}/
      - name: NME_{{ item }}
        add_NME: true
        brd_select: /model_1/A/{{ item +1 }}/
    with_items: residues

警告

Note that when using templates, especially when operating within a template, there is a distinction between strings and numbers.

タスク(tasks)セクション

タスクセクションは次の3つのパートに分かれています。

  • 条件

  • タスク処理内容

  • フレーム分子の定義

条件

with_items

タスク処理を繰り返します。

when

条件を満足した場合にタスク処理を実行します。

include

外部ファイルを挿入します。

tasks:
  - name: include_step1
    include_tasks: "step1.yaml"

タスク処理内容

mail

電子メールを送信します。

- name: send_mail
  mail:
    smtp_server: xxx.xxx.xxx.xxx
    smtp_port: xx
    use_SSL: yes
    smtp_account: xxxx
    smtp_password: xxxx
    from_address: xxxx@xxxx.xxxx
    to_address: xxxx@xxxx.xxxx
    subject: 'test mail'
    msg: 'job finished.'

debug

デバッグ用に出力します。

frame

フレーム分子の定義

名前(必須)

全てのフレーム分子は名前が必要です。この名前に応じて作業ディレクトリがカレントディレクトリの下に作成されます。

cmd_alias

Accept values of the dictionary type. Replaces the default external commands. The following commands are currently supported:

  • archive

  • mat-extend

  • mat-mul

  • mat-select

  • mat-symetrize

  • mat-transpose

  • mat-diagonal

fragments

フレーム分子は1つ以上のフラグメントから構成されます。フラグメントは以下のキーワードによって定義されます。全てのフラグメントはフラグメントの名前として利用する "name" 属性が必要です。

  • atomlist

atomlist は原子のリストからフラグメントを作成します。

- name: N2
  sp: true
  fragments:
    - name: N2
      atomlist:
        - "N  0.000000   0.000000   0.000000"
        - "N  1.000000   0.000000   0.000000"

atomlist は配列オブジェクトである必要があります。また、それぞれの原子はスペースで区切られるか、以下のような配列オブジェクトである必要があります。

- name: OH
  sp: true
  fragments:
    - name: OH
      atomlist:
        - [O, -7.328, -30.909,  17.923]
        - [H, -6.026, -31.757,  17.909]

  • add_CH3

    このキーワードが yes で定義された場合、メチル基がフラグメントに付加されます。

    • displacement

      この原子がメチル基炭素と置換されます。この値はbridgeフォーマットのパスで指定します。

    • root

      置換される炭素原子の隣の原子を指定します。この値はbridgeフォーマットのパスで指定します。

      - name: small_mol
  sp: true
  fragments:
    - name: frag1
      add_CH3: true
      displacement: "/model_1/A/100/100_C1"
      root: "/model_1/A/100/100_C2"

  • add_ACE

アセチル基を付加してフラグメントに追加します。

- name: res_3
  sp: true
  fragments:
    - name: res_3
      brd_select: /model_1/A/3/
    - name: ACE_3
      add_ACE: true
      brd_select: /model_1/A/2/

  • add_NME

N-メチル基を付加してフラグメントに追加します。

- name: res_3
  sp: true
  fragments:
    - name: res_3
      brd_select: /model_1/A/3/
    - name: NME_3
      add_NME: true
      brd_select: /model_1/A/4/

  • reference

以前の計算結果を使ってフラグメントを作成します。

  • frame

    フレーム分子の名前を指定します。

  • fragment

    フラグメント名を指定します。

- name: frame_3
  sp: true
  fragments:
    - name: AA_3
      brd_select: /model_1/A/3/
    - name: ACE_3
      add_ACE: true
      brd_select: /model_1/A/3/
    - name: NME_3
      add_NME: true
      brd_select: /model_1/A/3/

- name: frame_3-7
  sp: true
  guess: QCLO
  fragments:
    - name: AA_3
      reference:
        frame: frame_3
        fragment: AA_3

  • brd_select

指定されたbridgeフォーマットのパスにある原子(団)をフラグメントに追加します。

フレーム分子に対する処理

以下のキーワードによって、対象のフレーム分子の処理を指示できます。

  • pre_scf

    pre_scf に yes が与えられると、SCF繰り返し計算前の処理が行われます。

  • guess

    Creation of the initial guess is executed. How to create guess depends on the value of "guess" keyword. Use mapping format in "guess" section.

    • 計算方法

      Specified method for guess. Possible values are as follows.

      • harris

        初期値をHarrisの汎関数によって求めます。この計算方法はデフォルトです。

      • QCLO

        初期値を対応するフラグメントからQCLO法によって求めます。子フラグメントのQCLOが作成されていなければ、自動的に計算します。

    • force

      boolean. The default value is False. If you want to force execution even if it has already been done, specify True.

  • sp

    sp に yes が与えられると、フレーム分子の一点計算を行います``pre_scf`` と guess キーワードが指定されていない場合、これらの処理は自動的に行われます。

  • force

    このキーワードが yes で定義された場合、メチル基がフラグメントに付加されます。

    - name: res_3-7
  pre_scf: yes
  guess: QCLO
  sp: yes
  force: yes

  • summary

    計算の概要を表示します。与えられた値によって3つの出力形式があります。

    • ブール値

      Trueが与えられると、標準的な内容を出力します。

      - name: res_3-7
  pre_scf: yes
  guess: QCLO
  sp: yes
  summary: yes

    • フォーマット文字列

      与えられたフォーマット文字列に従って出力します。特殊文字列は対応する内容に置換されます。

      キーワード

      説明

      {NUM_OF_ATOMS}

      原子数

      {NUM_OF_AO}

      原子軌道の総数

      {NUM_OF_MO}

      分子軌道の総数

      {METHOD}

      計算方法

      {IS_CONVERGED}

      SCF計算が収束したかどうか

      {ITERATION}

      SCF繰り返し回数

      {TOTAL_ENERGY}

      全エネルギー

      {GRADIENT_RMS}

      エネルギー微分のRMS

      		 
      - name: res_3-7
pre_scf: yes
guess: QCLO
sp: yes
summary: "atoms: {NUM_OF_ATOMS} iterations: {ITERATION}"

    • 辞書

      もしファイルに出力したい場合は、この方式を利用します。出力ファイルは追記モードで開かれます。

      • format

        フォーマット文字列を指定します。

      • filepath

        ファイルの出力先を指定します。

        - name: res_3-7
  pre_scf: yes
  guess: QCLO
  sp: yes
  summary:
    format: "atoms: {NUM_OF_ATOMS} iterations: {ITERATION}"
    filepath: "summary.txt"

default フレーム

default という名前は特別です。

"default" という名前で定義されたフレーム分子は、以後のデフォルトのパラメータとして利用されます。

基底関数セットとして DZVP2 、交換相関汎関数として B3LYP を用いた例を以下に示します。

tasks:
  - name: default
    brd_file: sample.brd
    basis_set: DZVP2
    XC_functional: B3LYP

  - name: N2
    atomlist:
      - "N  0.000000   0.000000   0.000000"
      - "N  1.000000   0.000000   0.000000"
